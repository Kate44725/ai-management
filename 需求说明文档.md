# AI算力管理平台需求说明文档

**版本**: V1.0
**日期**: 2026年3月
**作者**: 产品经理

---

## 1. 项目背景

### 1.1 项目目标

半导体数字化与IT装备部AI算力管理平台旨在统一管理红区和黄区两个网络区域的AI算力资源，提供算力申请、令牌分配、运营看板等功能，实现AI算力资源的高效利用和精细化管理。

### 1.2 项目定位

本平台面向半导体企业的IT管理员、项目负责人、运营人员及普通研发人员，提供一个可视化、交互式的AI算力管理门户。

### 1.3 预期用户

| 用户角色 | 使用场景 |
|---------|---------|
| 管理员 | 配额审批、用户额度管理、项目管理、模型管理、全局配置 |
| 项目负责人 | 查看本项目算力使用情况、管理项目成员、申请额度、本项目配额审批、批量调整本项目人员配额 |
| 运营人员 | 运营看板数据查看、数据导出分析、监控告警 |
| 普通研发人员 | 查看个人算力使用、申请额度、管理令牌 |

## 3. 非功能需求

### 3.1 性能要求

- 页面加载时间不超过3秒
- 图表渲染时间不超过1秒
- 数据导出响应时间不超过5秒

### 3.2 兼容性要求

- 支持主流浏览器：Chrome、Edge、Firefox、Safari
- 屏幕分辨率适配：1024x768及以上

### 3.3 安全要求

- 令牌信息部分隐藏展示
- 角色权限控制（普通用户/管理员）
- 操作日志记录

### 3.4 可用性要求

- 统一的Toast通知反馈
- 模态框确认操作
- 响应式布局适配

## 4. 数据模型

### 4.1 核心实体

| 实体 | 说明 |
|-----|------|
| 用户(USERS) | 平台用户，含姓名、部门、角色 |
| 项目(PROJECTS) | 算力使用项目，含配额、成员 |
| 令牌(TOKENS) | 用户API令牌，含名称、ID、状态 |
| 额度请求(QUOTA_REQUESTS) | 额度申请记录 |
| 模型(MODELS) | AI模型配置，含厂商、版本、状态 |
| 使用日志(USE_LOGS) | Token使用记录 |

### 4.2 演示数据

- 10个AI模型（GLM4.7、GPT-4、Claude3等）
- 10个部门（芯片设计部、验证部、研发部等）
- 20台GPU服务器，200张GPU
- 多名演示用户及项目

## 5. 界面原型（文字描述）

### 5.1 整体布局

- **左侧导航**: 固定侧边栏，包含主导航菜单
- **顶部栏**: 页面标题
- **主体区域**: 内容展示区，含卡片、表格、图表

### 5.2 关键交互

| 页面 | 交互元素 |
|-----|---------|
| 我的算力 | 令牌创建弹窗、项目切换弹窗 |
| 额度管理 | Tab切换、审批操作、批量选择 |
| 运营看板 | 维度按钮、时间选择器、图表切换 |
| 模型管理 | 厂商/状态按钮筛选、启用禁用开关 |
| 项目管理 | 项目编辑弹窗、成员管理弹窗 |
