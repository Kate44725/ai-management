# AI算力管理系统 - 需求规格说明书

> 版本：v1.0
> 日期：2026-03-09

## 1. 项目概述

### 1.1 项目背景

半导体数字化与IT装备部（四级部门）部署了模型服务（NEW API），提供给三级部门下的其他四级部门使用。用户的AI算力使用需要按照Token消耗分摊到各个项目。

### 1.2 项目目标

1. 建立统一的AI算力管理平台，实现Token消耗按项目分摊
2. 提供额度管理功能，支持项目经理分配额度
3. 提供运营分析功能，支持费用结算和统计分析
4. 提供统一API接口，支持多工具平台（IDE、仿真工具等）接入

### 1.3 系统架构

```
┌─────────────────────────────────────────────────────────────────┐
│                    系统二：AI算力管理系统                          │
│   额度管理 │ 运营分析 │ 需求流程 │ 用户入口                       │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│                    系统一：NEW API (开源大模型服务)                 │
└─────────────────────────────────────────────────────────────────┘
                              ▲
                              │ 调用API
┌─────────────────────────────────────────────────────────────────┐
│               系统三：SuperIDE/工具平台                           │
└─────────────────────────────────────────────────────────────────┘
```

## 2. 术语定义

| 术语 | 定义 |
|------|------|
| Token | 大语言模型API调用计量单位 |
| 项目 | 业务项目，承担AI算力费用 |
| 额度 | 项目分配给用户的Token配额 |
| Key | API调用凭证，关联用户、项目、工具 |
| 工具 | 使用AI能力的平台类型（IDE/仿真工具/Web） |

## 3. 需求假设

### 3.1 组织架构

| 假设 | 内容 |
|------|------|
| H1.1 | 模型服务只提供给同三级部门下的其他四级部门 |
| H1.2 | 使用方开发人员可能来自一级部门下的任何层级部门 |
| H1.3 | 组织架构：集团 → 一级部门 → 二级部门 → 三级部门 → 四级部门 |

### 3.2 项目与额度

| 假设 | 内容 |
|------|------|
| H2.1 | "项目"指业务项目，该项目承担AI算力费用 |
| H2.2 | 项目是动态变化的（会新建、解散） |
| H2.3 | 一个用户最多属于3个项目 |
| H2.4 | 用户额度由项目经理分配 |
| H2.5 | 费用直接分摊到项目 |

### 3.3 用户使用场景

| 假设 | 内容 |
|------|------|
| H3.1 | 约90%用户只使用1个项目的资源 |
| H3.2 | 约10%用户使用多个项目的资源 |
| H3.3 | 项目资源严格分开（涉及费用分摊） |

### 3.4 技术约束

| 假设 | 内容 |
|------|------|
| H4.1 | 系统一（NEW API）使用开源代码部署，原则上不修改源码 |
| H4.2 | 系统一和系统二数据库共享，系统二可直接查询Token使用量 |
| H4.3 | 系统三有统一身份认证机制 |
| H4.4 | Key粒度：用户 + 项目 + 工具类型 |

## 4. 方案对比

### 4.1 三个方案对比

| 对比项 | 方案A | 方案B | 方案C |

├── user_id

├── email
├── department_id

项目表 (projects)
├── project_id
├── project_name
├── department_id
├── budget_token

├── key_id

├── project_id
├── tool_type

├── record_id
├── key_id
├── token_count
├── timestamp
└── model_name

├── project_id
├── quota_amount
